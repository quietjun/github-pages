☕ Java 멀티스레드 완전 가이드

소개 → 기초 → 고급 → Virtual Thread까지 단계별 학습

1. Thread 개요

Thread vs Process

  • Process: 독립적 메모리, 격리된 실행 환경
  • Thread: 같은 프로세스의 메모리 공유, 경량
  • Thread 100개 생성 = Process 1개 생성보다 빠름 & 가벼움

멀티스레드의 필요성

  • 응답성 향상: UI 블록 방지 (네트워크 요청 중에도 UI 반응)
  • 효율성: 멀티코어 활용, 동시 작업 처리
  • 아키텍처: 웹 서버 (1 thread per client) 구현

Thread 생성 방식

1️⃣ Thread 클래스 상속
class MyThread extends Thread { public void run() { ... } } new MyThread().start();
2️⃣ Runnable 구현 (권장)
class MyTask implements Runnable { public void run() { ... } } new Thread(new MyTask()).start();

⚠️ start() vs run()

  • run(): 호출 스레드에서 실행 (멀티스레드 X)
  • start(): 새 스레드 생성 후 run() 실행 (멀티스레드 O)
  • 항상 start()를 호출하세요!

2. Thread 기초 개념

Main Thread

  • 프로그램 시작 시 자동으로 생성
  • main() 메서드 실행
  • 모든 자식 스레드가 종료되면 프로그램 종료
  • Thread.currentThread() → main thread 정보

Thread 속성

속성 설명 기본값
ID 각 스레드의 고유 번호 자동 할당
Name 디버깅용 이름 Thread-N
Priority 스케줄 우선순위 (1~10) 5 (NORM)
Daemon 백그라운드 실행 여부 false

Daemon Thread

  • 메인 스레드가 종료되면 즉시 종료
  • 보통 백그라운드 작업 (GC, 타이머)
  • setDaemon(true) → start() 전에 호출
  • ⚠️ 중요한 데이터 저장에는 사용 금지

Thread Priority

  • MIN_PRIORITY (1) → 최저
  • NORM_PRIORITY (5) → 기본 (대부분 사용)
  • MAX_PRIORITY (10) → 최고
  • setPriority()로 설정
  • ⚠️ OS 의존적, 보장 안 됨

Thread 정보 조회

Thread t = Thread.currentThread(); System.out.println("ID: " + t.getId()); System.out.println("Name: " + t.getName()); System.out.println("Priority: " + t.getPriority()); System.out.println("State: " + t.getState()); System.out.println("Alive: " + t.isAlive());

3. Thread 생명주기

Thread 상태 다이어그램

5가지 상태

상태 설명
NEW 객체 생성, start() 호출 전
RUNNABLE 실행 대기 중 (CPU 시간 할당 기다리는 중)
RUNNING CPU 할당받아 실행 중 (RUNNABLE에 포함)
WAITING 다른 스레드의 신호 대기 (무한정)
TIMED_WAITING 시간 제한이 있는 대기 (sleep, wait(ms))
BLOCKED Lock 획득 대기 중
TERMINATED 실행 완료, 스레드 종료

상태 전이 핵심 메서드

  • start(): NEW → RUNNABLE
  • sleep(ms): RUNNABLE → TIMED_WAITING → RUNNABLE
  • wait(): RUNNABLE → WAITING (lock 해제)
  • notify() / notifyAll(): WAITING → RUNNABLE
  • join(): 다른 스레드 완료 대기
  • interrupt(): 중단 신호 (대기 깨우기)

⚠️ Thread 중단 메커니즘 (Graceful Shutdown)

핵심: 강제 중단 불가능

  • stop() 안 됨: deprecated (데이터 오염 위험)
  • suspend() 안 됨: deprecated (데드락 위험)
  • Running 중인 스레드: 외부에서 강제 중단 불가
  • 해결책: 스레드가 자신의 interrupt 상태를 확인

두 가지 중단 방식

1️⃣ interrupt() 사용 (표준)
Thread t = new Thread(() -> { while (!Thread.currentThread() .isInterrupted()) { doWork(); } }); t.start(); t.interrupt(); // 중단 신호
  • wait/sleep 중: 즉시 깨어남
  • Running 중: isInterrupted() 확인 필수
  • 표준 Java 방식
2️⃣ volatile 플래그 (명시적)
volatile boolean shouldStop = false; Thread t = new Thread(() -> { while (!shouldStop) { doWork(); } }); t.start(); shouldStop = true; // 중단 신호
  • 응답성 제어 가능
  • 명확한 의도
  • 메모리 가시성 보장

Running 중 interrupt 처리

// ❌ interrupt()만으로는 부족 Thread t = new Thread(() -> { while (true) { heavyComputation(); // 1초 소요 // interrupt 신호를 확인 안 함! } }); // ✅ 주기적으로 확인 필요 Thread t = new Thread(() -> { while (!Thread.currentThread() .isInterrupted()) { for (long i = 0; i < 1_000_000_000L; i++) { if (i % 10_000_000 == 0 && Thread.currentThread() .isInterrupted()) { return; // 즉시 종료 } } } });

Thread.sleep() vs wait()

구분 sleep() wait()
Lock 해제 ❌ 유지 ✅ 해제
호출 위치 어디든 가능 synchronized 블록
깨우는 방법 시간 경과 자동 notify() 필요

4. Thread 동기화

Race Condition

⚠️ 동기화가 필요한 이유

int count = 0; thread1: count++ // Read 10, Write 11 thread2: count++ // Read 10, Write 11 // Expected: 12, Actual: 11
  • count++ 는 원자적(atomic) 연산이 아님
  • Read → Modify → Write 3단계로 분해
  • 여러 스레드가 동시에 접근 시 예측 불가

synchronized 키워드

메서드 동기화
synchronized void increment() { count++; }
  • 객체의 lock 자동 취득
  • 메서드 진입 시 lock
  • 메서드 종료 시 lock 해제
블록 동기화
synchronized (this) { count++; }
  • 더 세밀한 제어
  • lock 범위 최소화
  • 성능 최적화

Lock과 Monitor

  • 모든 Java 객체 = Monitor 보유
  • 한 번에 한 스레드만 lock 획득
  • Intrinsic Lock (암묵적 lock)
  • Mutual Exclusion (상호 배제) 보장

java.util.concurrent.locks

종류 특징
ReentrantLock 재진입 가능, 명시적 lock/unlock, 공정성 옵션
ReadWriteLock 읽기는 동시, 쓰기는 배타적
StampedLock 낙관적 읽기 (stamp), 초고성능

ReentrantLock 예제

ReentrantLock lock = new ReentrantLock(); lock.lock(); try { count++; } finally { lock.unlock(); }

Volatile 키워드

  • 변수의 가시성 보장
  • 메인 메모리에서 항상 읽음
  • 원자성은 보장 ❌ (count++ 여전히 위험)
  • 신호 변수, 플래그에 적합

AtomicInteger / AtomicLong

AtomicInteger count = new AtomicInteger(0); count.incrementAndGet(); // 원자적 연산 count.getAndAdd(5); count.compareAndSet(10, 20);

5. ExecutorService & 스레드풀

왜 스레드풀인가?

  • ❌ 태스크마다 new Thread(): 생성 비용 높음
  • ✅ 미리 생성된 스레드 재사용: 빠름 & 저비용
  • 스레드 개수 제한 → 시스템 안정성
  • 웹 서버, 비동기 작업 표준

Executors 팩토리

유형 생성 방식 용도
newFixedThreadPool(n) n개 스레드 고정 I/O 작업, 일반 비동기
newCachedThreadPool() 필요하면 생성, 유휴면 재사용 짧은 비동기 작업
newSingleThreadExecutor() 1개 스레드만 순서 보장 필요시
newScheduledThreadPool(n) 일정 시간 후 실행 스케줄링, 타이머

ExecutorService 기본 사용법

ExecutorService executor = Executors.newFixedThreadPool(4); // 1. submit(Runnable) executor.submit(() -> { System.out.println("Task 1"); }); // 2. submit(Callable) Future future = executor.submit(() -> { return 42; }); // 3. 결과 조회 Integer result = future.get(); // 블로킹 // 4. shutdown executor.shutdown();

Future & Callback

  • future.get(): 결과 대기 (블로킹)
  • future.isDone(): 완료 여부 확인
  • future.cancel(): 실행 취소
  • future.get(timeout, unit): 제한시간 대기

invokeAll / invokeAny

// invokeAll: 모든 작업 완료 대기 List> futures = executor.invokeAll(tasks); // invokeAny: 가장 먼저 완료된 것 반환 String result = executor.invokeAny(tasks);

ThreadPoolExecutor 세부 설정

ThreadPoolExecutor pool = new ThreadPoolExecutor( 2, // corePoolSize 5, // maxPoolSize 60, TimeUnit.SECONDS, // keepAliveTime new LinkedBlockingQueue<>() // queue );

스레드풀 사이즈 가이드

  • CPU 집약적: corePoolSize = CPU 코어 수
  • I/O 집약적: corePoolSize = 2 ~ 3 × 코어 수
  • 처음부터 완벽한 값 → 성능 테스트 후 조정

6. 모던 자바 (Java 8+)

CompletableFuture

  • Future의 한계 극복 (get() 블로킹 강제)
  • 콜백 기반 비동기 처리
  • 여러 async 작업 조합 가능
  • 예외 처리 용이

CompletableFuture 기본

// 1. 비동기 실행 CompletableFuture future = CompletableFuture.supplyAsync(() -> { return 42; }); // 2. 콜백 등록 future.thenAccept(result -> System.out.println(result)); // 3. 체이닝 future .thenApply(x -> x * 2) .thenAccept(System.out::println);

비동기 조합

메서드 설명
thenApply(f) 결과 변환
thenAccept(f) 결과 소비 (반환 없음)
thenRun(r) 입력 무시하고 실행
thenCompose(f) 비동기 함수 체이닝 (flatMap)
thenCombine(f) 두 결과 조합

여러 작업 병렬 처리

CompletableFuture f1 = CompletableFuture.supplyAsync(() -> "A"); CompletableFuture f2 = CompletableFuture.supplyAsync(() -> "B"); // 모두 완료 대기 CompletableFuture.allOf(f1, f2) .thenAccept(v -> System.out.println("Done")); // 먼저 완료된 것 사용 CompletableFuture.anyOf(f1, f2) .thenAccept(System.out::println);

예외 처리

CompletableFuture.supplyAsync(() -> 42) .thenApply(x -> x / 0) // ArithmeticException .exceptionally(ex -> { System.err.println("Error: " + ex); return -1; }) .thenAccept(System.out::println);

Parallel Stream

// 순차 처리 (느림) list.stream() .map(item -> heavyComputation(item)) .collect(Collectors.toList()); // 병렬 처리 (빠름) list.parallelStream() .map(item -> heavyComputation(item)) .collect(Collectors.toList());

⚠️ parallelStream 주의

  • ForkJoinPool 사용 (공유 스레드풀)
  • CPU 집약적 작업에만 유리
  • I/O 대기 작업 부적합 (블로킹 발생)
  • 상태 공유 금지 (synchronized 비용 높음)

Reactive Stream

  • Project Reactor, RxJava: reactive 라이브러리
  • BackPressure 지원 (생산자-소비자 속도 조절)
  • 구독자(Subscriber) 기반 이벤트 처리
  • Spring WebFlux 기반

7. Virtual Thread (Java 21+)

문제: Platform Thread의 한계

  • OS Thread 1:1 매핑 → 생성 비용 높음
  • 메모리: 1 platform thread ≈ 2MB
  • 10,000 스레드 = 20GB 메모리! 💾
  • Context switching 비용 증가

해결책: Virtual Thread

  • JVM 관리하는 경량 스레드
  • 메모리: 1 virtual thread ≈ 100~200bytes
  • 100만 개 virtual thread = 200MB 메모리! 🚀
  • OS Thread 수는 적음 (CPU 코어 수 정도)
  • JVM이 scheduling 담당 (자동 load balancing)

Virtual Thread vs Platform Thread

구분 Platform Thread Virtual Thread
메모리 2MB 100~200bytes
생성 비용 비쌈 매우 저렴
생성 한계 수천 개 백만 개 이상 가능
Context Switching OS 레벨 JVM 레벨
I/O 블로킹 스레드 블록 자동 스케줄링 (Carrier thread 전환)

Virtual Thread 생성

// Java 21+ // 1. Thread.ofVirtual() Thread vthread = Thread.ofVirtual() .name("vt-1") .start(() -> { System.out.println("Virtual thread"); }); // 2. 간편 방법 Thread vthread2 = Thread.startVirtualThread(() -> { System.out.println("Virtual thread"); }); // 3. ExecutorService로 관리 ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();

Virtual Thread 특성

  • 항상 daemon thread ❌ (제어 가능)
  • Priority 무시 (모두 NORM_PRIORITY)
  • ThreadGroup 지원
  • Thread local (대비: 메모리 누수 위험 증가)

Virtual Thread는 만능인가?

⚠️ Virtual Thread 부적합한 경우

  • CPU 집약적 작업 (장시간 계산)
  • synchronized 블록 (Carrier thread 고착)
  • ReentrantLock 등 명시적 lock (권장: java.util.concurrent.locks 피함)

Structured Concurrency (Preview)

try (var scope = new StructuredTaskScope()) { var future1 = scope.fork(() -> task1()); var future2 = scope.fork(() -> task2()); scope.joinUntil(Instant.now().plus(Duration.ofSeconds(5))); return future1.resultNow() + " " + future2.resultNow(); }

Virtual Thread 도입 가이드

  • ✅ 기존 코드 그대로 사용 (ThreadPool → newVirtualThreadPerTaskExecutor)
  • ✅ I/O 블로킹 작업: 즉시 도입
  • ⚠️ 고성능 튜닝 필요: 마이그레이션 후 재측정
  • ❌ synchronized 로직 많으면: 리팩토링 필수

8. 실습 예제

1️⃣ 기본 Thread 생성

문제

3개 스레드에서 각각 1~5를 순서대로 출력하세요.

public class BasicThread { public static void main(String[] args) throws InterruptedException { Thread t1 = new Thread(() -> { for (int i = 1; i <= 5; i++) { System.out.println("T1: " + i); } }); Thread t2 = new Thread(() -> { for (int i = 1; i <= 5; i++) { System.out.println("T2: " + i); } }); t1.start(); t2.start(); t1.join(); // T1 완료 대기 t2.join(); // T2 완료 대기 System.out.println("Main 종료"); } }

2️⃣ Synchronized - Race Condition 해결

문제

2개 스레드에서 count++를 1000번씩 실행. 최종값이 2000이 되도록 하세요.

public class CounterSync { static int count = 0; synchronized static void increment() { count++; } public static void main(String[] args) throws InterruptedException { Thread t1 = new Thread(() -> { for (int i = 0; i < 1000; i++) { increment(); } }); Thread t2 = new Thread(() -> { for (int i = 0; i < 1000; i++) { increment(); } }); t1.start(); t2.start(); t1.join(); t2.join(); System.out.println("count = " + count); // 2000 } }

3️⃣ Producer-Consumer 패턴

문제

Producer가 1~10을 큐에 넣고, Consumer가 꺼낸다. 동기화 필요.

public class ProducerConsumer { static class Queue { private int[] buffer = new int[1]; private int count = 0; synchronized void produce(int value) throws InterruptedException { while (count == 1) { wait(); // 버퍼 가득차면 대기 } buffer[0] = value; count++; System.out.println("Produce: " + value); notifyAll(); // Consumer 깨우기 } synchronized int consume() throws InterruptedException { while (count == 0) { wait(); // 버퍼 비어있으면 대기 } int value = buffer[0]; count--; System.out.println("Consume: " + value); notifyAll(); // Producer 깨우기 return value; } } public static void main(String[] args) { Queue queue = new Queue(); Thread producer = new Thread(() -> { try { for (int i = 1; i <= 10; i++) { queue.produce(i); Thread.sleep(100); } } catch (InterruptedException e) {} }); Thread consumer = new Thread(() -> { try { for (int i = 0; i < 10; i++) { queue.consume(); Thread.sleep(200); } } catch (InterruptedException e) {} }); producer.start(); consumer.start(); } }

4️⃣ ExecutorService 활용

문제

10개 작업을 4개 스레드로 처리하고, 모든 작업 완료 대기.

public class ExecutorExample { public static void main(String[] args) throws InterruptedException { ExecutorService executor = Executors.newFixedThreadPool(4); List> futures = new ArrayList<>(); for (int i = 1; i <= 10; i++) { final int taskId = i; Future future = executor.submit(() -> { System.out.println( "Task " + taskId + " on " + Thread.currentThread().getName()); return taskId * 2; }); futures.add(future); } // 모든 작업 완료 대기 for (Future future : futures) { Integer result = future.get(); System.out.println("Result: " + result); } executor.shutdown(); System.out.println("모든 작업 완료"); } }

5️⃣ CompletableFuture 비동기 조합

문제

2개의 비동기 작업을 병렬 실행 후 결과 조합.

public class CompletableFutureExample { static int fetchUserData(int userId) throws InterruptedException { Thread.sleep(1000); return userId * 100; } static int fetchOrderData(int userId) throws InterruptedException { Thread.sleep(1000); return userId * 10; } public static void main(String[] args) throws Exception { long start = System.currentTimeMillis(); CompletableFuture user = CompletableFuture.supplyAsync( () -> { try { return fetchUserData(1); } catch (InterruptedException e) { return 0; } } ); CompletableFuture order = CompletableFuture.supplyAsync( () -> { try { return fetchOrderData(1); } catch (InterruptedException e) { return 0; } } ); user.thenCombine(order, (u, o) -> { System.out.println( "User: " + u + ", Order: " + o); return u + o; }).thenAccept(total -> System.out.println("Total: " + total) ).join(); long elapsed = System.currentTimeMillis() - start; System.out.println( "소요시간: " + elapsed + "ms"); // ~1000ms } }

6️⃣ Virtual Thread 활용 (Java 21+)

문제

100,000개 가상 스레드에서 HTTP 요청 시뮬레이션.

public class VirtualThreadExample { static void simulateHttpRequest(int id) throws InterruptedException { System.out.println("Request " + id + " on " + Thread.currentThread()); Thread.sleep(100); // I/O 시뮬레이션 } public static void main(String[] args) throws Exception { long start = System.currentTimeMillis(); // Virtual thread per task ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor(); List> futures = new ArrayList<>(); for (int i = 1; i <= 100_000; i++) { final int id = i; futures.add(executor.submit(() -> { try { simulateHttpRequest(id); } catch (InterruptedException e) {} })); } // 모든 요청 완료 대기 executor.shutdown(); executor.awaitTermination( 10, TimeUnit.SECONDS); long elapsed = System.currentTimeMillis() - start; System.out.println( "완료: " + elapsed + "ms"); } }
