1. Thread 개요

Thread vs Process

  • Process: 독립적 메모리, 격리된 실행 환경
  • Thread: 같은 프로세스의 메모리 공유, 경량
  • Thread 100개 생성 = Process 1개 생성보다 빠름 & 가벼움

Thread 생성 방식

1️⃣ Thread 클래스 상속
class MyThread extends Thread { public void run() { ... } } new MyThread().start();
2️⃣ Runnable 구현 (권장)
class MyTask implements Runnable { public void run() { ... } } new Thread(new MyTask()).start();

⚠️ start() vs run()

  • run(): 호출 스레드에서 실행 (멀티스레드 X)
  • start(): 새 스레드 생성 후 run() 실행 (멀티스레드 O)
  • 항상 start()를 호출하세요!

2. Thread 기초

Main Thread

  • 프로그램 시작 시 자동으로 생성
  • main() 메서드 실행
  • 모든 자식 스레드가 종료되면 프로그램 종료

Thread 속성

속성 설명 기본값
ID 각 스레드의 고유 번호 자동 할당
Name 디버깅용 이름 Thread-N
Priority 스케줄 우선순위 (1~10) 5 (NORM)
Daemon 백그라운드 실행 여부 false

Daemon Thread

  • 메인 스레드가 종료되면 즉시 종료
  • 보통 백그라운드 작업 (GC, 타이머)
  • setDaemon(true) → start() 전에 호출

3. Thread 생명주기

5가지 상태

상태 설명
NEW 객체 생성, start() 호출 전
RUNNABLE 실행 대기 중 (CPU 시간 할당 기다리는 중)
WAITING 다른 스레드의 신호 대기 (무한정)
TIMED_WAITING 시간 제한이 있는 대기 (sleep, wait(ms))
BLOCKED Lock 획득 대기 중
TERMINATED 실행 완료, 스레드 종료

상태 전이 핵심 메서드

  • start(): NEW → RUNNABLE
  • sleep(ms): RUNNABLE → TIMED_WAITING → RUNNABLE
  • wait(): RUNNABLE → WAITING (lock 해제)
  • notify(): WAITING → RUNNABLE
  • join(): 다른 스레드 완료 대기
  • interrupt(): 중단 신호

⚠️ Thread 중단 (Graceful Shutdown)

강제 중단 불가능

  • stop() 안 됨: deprecated (데이터 오염)
  • Running 중: 외부에서 강제 중단 불가
  • 해결책: 스레드가 interrupt 상태를 주기적으로 확인

두 가지 중단 방식

1️⃣ interrupt() (표준)
while (!Thread.currentThread() .isInterrupted()) { try { Thread.sleep(1000); } catch (InterruptedException e) { break; } }
2️⃣ volatile 플래그
volatile boolean shouldStop = false; while (!shouldStop) { try { Thread.sleep(1000); } catch (InterruptedException e) { break; } }

4. Thread 동기화

Race Condition

⚠️ 동기화가 필요한 이유

int count = 0; thread1: count++ // Read 10, Write 11 thread2: count++ // Read 10, Write 11 // Expected: 12, Actual: 11
  • count++ 는 원자적(atomic) 연산이 아님
  • Read → Modify → Write 3단계로 분해
  • 여러 스레드가 동시에 접근 시 예측 불가

synchronized 키워드

메서드 동기화
synchronized void increment() { count++; }
블록 동기화
synchronized (this) { count++; }

java.util.concurrent.locks

종류 특징
ReentrantLock 재진입 가능, 명시적 lock/unlock
ReadWriteLock 읽기는 동시, 쓰기는 배타적
StampedLock 낙관적 읽기, 초고성능

Volatile 키워드

  • 변수의 가시성 보장
  • 메인 메모리에서 항상 읽음
  • 원자성은 보장 ❌ (count++ 여전히 위험)
  • 신호 변수, 플래그에 적합

AtomicInteger / AtomicLong

AtomicInteger count = new AtomicInteger(0); count.incrementAndGet(); // 원자적 연산 count.getAndAdd(5); count.compareAndSet(10, 20);

5. ExecutorService & 스레드풀

왜 스레드풀인가?

  • ❌ 태스크마다 new Thread(): 생성 비용 높음
  • ✅ 미리 생성된 스레드 재사용: 빠름 & 저비용
  • 스레드 개수 제한 → 시스템 안정성

Executors 팩토리

유형 생성 방식 용도
newFixedThreadPool(n) n개 스레드 고정 I/O 작업, 일반 비동기
newCachedThreadPool() 필요하면 생성, 유휴면 재사용 짧은 비동기 작업
newSingleThreadExecutor() 1개 스레드만 순서 보장 필요시

ExecutorService 기본 사용법

ExecutorService executor = Executors.newFixedThreadPool(4); // submit(Runnable) executor.submit(() -> { System.out.println("Task 1"); }); // submit(Callable) Future future = executor.submit(() -> 42); Integer result = future.get(); // 블로킹 // shutdown executor.shutdown();

Future & Callback

  • future.get(): 결과 대기 (블로킹)
  • future.isDone(): 완료 여부 확인
  • future.cancel(): 실행 취소

6. 모던 자바 (Java 8+)

CompletableFuture

  • Future의 한계 극복 (get() 블로킹 강제)
  • 콜백 기반 비동기 처리
  • 여러 async 작업 조합 가능

CompletableFuture 기본

// 1. 비동기 실행 CompletableFuture future = CompletableFuture.supplyAsync(() -> 42); // 2. 콜백 등록 future.thenAccept(result -> System.out.println(result)); // 3. 체이닝 future.thenApply(x -> x * 2) .thenAccept(System.out::println);

여러 작업 병렬 처리

CompletableFuture f1 = CompletableFuture.supplyAsync(() -> "A"); CompletableFuture f2 = CompletableFuture.supplyAsync(() -> "B"); // 모두 완료 대기 CompletableFuture.allOf(f1, f2) .thenAccept(v -> System.out.println("Done")); // 먼저 완료된 것 사용 CompletableFuture.anyOf(f1, f2) .thenAccept(System.out::println);

Parallel Stream

// 순차 처리 list.stream() .map(item -> heavyComputation(item)) .collect(Collectors.toList()); // 병렬 처리 list.parallelStream() .map(item -> heavyComputation(item)) .collect(Collectors.toList());

7. Virtual Thread (Java 21+)

문제: Platform Thread의 한계

  • OS Thread 1:1 매핑 → 생성 비용 높음
  • 메모리: 1 platform thread ≈ 2MB
  • 10,000 스레드 = 20GB 메모리! 💾

해결책: Virtual Thread

  • JVM 관리하는 경량 스레드
  • 메모리: 1 virtual thread ≈ 100~200bytes
  • 100만 개 virtual thread = 200MB 메모리! 🚀
  • I/O 블로킹 작업에 최적화

Virtual Thread vs Platform Thread

구분 Platform Thread Virtual Thread
메모리 2MB 100~200bytes
생성 비용 비쌈 매우 저렴
생성 한계 수천 개 백만 개 이상
I/O 블로킹 스레드 블록 자동 스케줄링

Virtual Thread 생성

// Java 21+ // 1. Thread.ofVirtual() Thread vthread = Thread.ofVirtual() .name("vt-1") .start(() -> { System.out.println("Virtual thread"); }); // 2. 간편 방법 Thread vthread2 = Thread.startVirtualThread(() -> { System.out.println("Virtual thread"); }); // 3. ExecutorService ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();
