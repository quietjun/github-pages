⚠️ 핵심 포인트
  • 비용 차이의 본질: 메모리 읽기(0.25) vs 디스크 I/O(1.00) 단가 차이
  • 옵티마이저는 테이블 크기 기반으로 버퍼 풀 캐시 여부를 추정 → 큰 테이블 = I/O 가정
  • Secondary 인덱스 테이블이 더 작거나 캐시 친화적이라고 추정된 결과
  • Node cost(0.1) ≠ Query cost(1.00): 노드는 서버 레벨 비용만, 쿼리 전체는 엔진(I/O) 비용 포함
  • 비용 = 옵티마이저의 추정값: 실제 실행 시간과 반드시 일치하지 않음
💡 실무 팁: EXPLAIN ANALYZE로 추정 비용(cost)과 실제 실행 시간(actual time)을 함께 확인하면 옵티마이저의 통계 정확도를 검증할 수 있음. 비용이 높아도 실제론 빠를 수 있음.
