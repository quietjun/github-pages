CSR에서 SEO 문제가 발생하는 이유 시각화

CSR 렌더링 흐름

→ 브라우저가 JS를 실행해 DOM을 동적으로 구성
→ 초기 HTML은 빈 껍데기(shell)만 존재

크롤러가 보는 것 vs 사용자가 보는 것

→ 동일한 URL이지만 수집 시점에 따라 완전히 다른 콘텐츠
→ 크롤러는 JS 실행 없이 초기 HTML만 파싱

브라우저 (사용자)

✅ JS 실행 후 완성된 DOM 확인
✅ 메타 태그 · 제목 · 본문 · 링크 모두 존재
✅ Open Graph / JSON-LD 로드됨
<head> <title>상품 상세 페이지</title> <meta name="description" content="최고의 제품..."/> </head> <body> <h1>상품명</h1> <p>상품 설명이 여기에...</p> <a href="/related">관련 상품</a> </body>

검색 크롤러 (Googlebot 등)

❌ JS 미실행 → 빈 div만 인식
❌ 메타 정보 없음 → 인덱싱 불가
❌ 링크 없음 → 크롤링 경로 단절
<head> <title></title> <!-- meta 없음 --> </head> <body> <div id="root"> <!-- JS가 채워줘야 함 --> </div> <script src="bundle.js"> </script> </body>

💡 Googlebot의 JS 실행?
→ Google은 2단계 인덱싱 적용: ① HTML 즉시 수집 → ② 이후 JS 렌더링 (수일~수주 지연)
→ 지연 기간 동안 색인 누락 · 순위 저하 발생 가능
→ 다른 크롤러(Naver, Bing 등)는 JS 실행 미지원인 경우 많음

CSR로 인한 SEO 문제 원인

📄
빈 <title> · <meta description>

초기 HTML에 메타 정보 없음 → 검색 결과 미노출 / 제목·설명 자동생성(부정확)

🔗
내부 링크 크롤링 단절

JS 실행 전 <a href> 없음 → 크롤러가 사이트 구조 파악 불가 · PageRank 전달 차단

⏱
렌더링 지연 → TTI / LCP 악화

JS 다운로드·파싱·실행 완료 전까지 콘텐츠 없음 → Core Web Vitals 점수 저하 → 순위 영향

🗺
동적 라우팅 인식 불가

React Router 등으로 관리되는 URL → HTML 단에서 경로 정보 없음 → 개별 페이지 인덱싱 어려움

📊
구조화 데이터(JSON-LD) 지연 로드

JS로 주입되는 Schema.org 마크업 → 크롤러 수집 시 부재 → 리치 스니펫 표시 안 됨

🤖
크롤 예산(Crawl Budget) 낭비

빈 페이지 반복 크롤 → 유의미한 콘텐츠 미수집 → 대규모 사이트에서 인덱싱 효율 저하

해결 전략

🏆 권장 접근법 (SEO 영향도 순)
최우선 SSR (Server-Side Rendering)

→ 서버가 완성된 HTML 반환 → 크롤러 즉시 수집 가능
→ Next.js getServerSideProps / Nuxt.js 활용

권장 SSG (Static Site Generation)

→ 빌드 타임에 HTML 생성 → 정적 콘텐츠에 최적
→ Next.js getStaticProps / Gatsby / Astro

보조 수단 Pre-rendering / Dynamic Rendering

→ 크롤러 감지 후 미리 렌더된 HTML 제공
→ Rendertron · Prerender.io 활용 (임시방편)

필수 병행 React Helmet / next/head

→ SSR·SSG와 함께 메타 태그 서버 주입 필수
→ title · description · OG · canonical 모두 포함

📌 CSR 유지 시 최소한의 SEO 조치
🔍 sitemap.xml 제출
📜 robots.txt 최적화
🏷 정적 메타 태그 삽입
⚡ 번들 크기 최소화
🗂 구조화 데이터 SSR 주입
