Kubernetes 아키텍처

마이크로서비스 배포 및 오케스트레이션 구조

1️⃣ 빌드 계층 (Build Layer)
Source Code → Dockerfile 정의
Build Process → Image 생성 & Runtime 패키징
Registry Push → Container Image를 레지스트리에 저장
↓
2️⃣ 배포 계층 (Deployment Layer)
Helm Chart → Kubernetes 리소스 템플릿화
ArgoCD / CI-CD → GitOps 기반 자동 배포
Registry Pull → 클러스터에서 Image 다운로드
↓
3️⃣ 클러스터 계층 (Cluster Layer)
Pod → 컨테이너 실행 단위 (app + sidecar)
Sidecar → 로깅, 모니터링, 트레이싱 (observability)
gRPC/HTTP → Pod 간 통신
ETCD → 클러스터 상태 저장소
↓
4️⃣ 서비스 계층 (Service & Networking)
ClusterIP → 클러스터 내부 통신 (기본값)
NodePort → 각 노드의 포트로 외부 접근 (테스트용)
LoadBalancer → 외부 로드밸런서 통합
ExternalDNS → DNS 자동 관리
💡 핵심 개념
Pod : 컨테이너 + Sidecar(로깅/모니터링) → 최소 배포 단위
Service : Pod 접근 방식 → ClusterIP(내부) / LoadBalancer(외부)
GitOps : Helm + ArgoCD → 선언형 배포 자동화
ETCD : 클러스터 상태 관리 → 마스터 노드의 핵심 저장소
