Model Context Protocol (MCP)

Spring AI — @Tool 방식과의 차이점부터 MCP 아키텍처,
Transport, 클라이언트·서버 구성까지의 완전 가이드

🔧 @Tool vs MCP — 무엇이 다른가
Spring AI의 두 가지 Tool 통합 방식. @Tool은 인프로세스, MCP는 아웃오브프로세스(프로토콜 기반) 연동.
@Tool Spring AI Native
📦 In-Process — 같은 JVM 내 실행
⚙️ 메서드에 @Tool 어노테이션 선언
🔗 Spring Bean으로 등록, 자동 연결
📝 스키마: 파라미터 타입 → 자동 생성
🚫 외부 시스템 공유 불가
🚫 언어 종속 (Java/Kotlin 전용)
MCP Anthropic 표준 프로토콜
🌐 Out-of-Process — 별도 프로세스/서버
📡 JSON-RPC 2.0 기반 통신
🔗 STDIO / SSE 두 가지 Transport
📝 스키마: JSON Schema로 명시 정의
✅ 여러 AI 에이전트에서 재사용
✅ 언어 독립 (Node, Python, Java 등)
비교 항목 @Tool 방식 MCP 방식
실행 위치 동일 JVM 내부 독립 프로세스 / 원격 서버
통신 방식 메서드 직접 호출 JSON-RPC 2.0 over STDIO/SSE
선언 방법 @Tool 어노테이션 McpServerFeatures.SyncToolSpecification
재사용성 ✗ 단일 앱 내부 ✓ 다중 클라이언트/에이전트
언어 독립성 ✗ Java/Kotlin 전용 ✓ 모든 언어 구현체
동적 갱신 ✗ 재기동 필요 ✓ 런타임 Tools 변경 알림
제공 기능 Tool(함수) 만 Tool + Resource + Prompt
적합한 상황 단순 앱 내 기능 호출 공유 서비스, 마이크로서비스 통합
복잡도 낮음 — 빠른 시작 중간 — 설정 필요
코드로 보는 차이
@Tool 방식
JAVA WeatherService.java
@Service public class WeatherService { @Tool(description = "날씨 조회") public String getWeather( @ToolParam String city ) { return weatherApi.fetch(city); } } // ChatClient에 직접 등록 chatClient.tools(weatherService);
⚠️ 이 서비스는 이 앱에서만 사용 가능. 다른 AI 에이전트에서 재활용하려면 코드 복사 필요.
MCP 방식
JAVA WeatherMcpServer.java
// MCP 서버 — 독립 프로세스로 기동 var tool = new McpServerFeatures .SyncToolSpecification( new Tool("get_weather", "날씨 조회", schema), req -> { String city = req.arguments() .get("city"); return callResult(fetch(city)); } ); // 어느 AI 클라이언트든 연결 가능 ✓
✅ MCP 서버 하나를 여러 Claude / GPT / Spring AI 클라이언트가 동시에 연결해서 사용.
🏗️ MCP 아키텍처 — Host / Client / Server
세 계층으로 구성. Host가 여러 MCP Client를 관리하고, 각 Client가 외부 MCP Server와 연결.
🏠
Host Application
AI 모델을 실행하는 메인 앱.
→ Spring Boot 앱 자체
→ 여러 MCP Client를 보유
→ ChatClient 통해 AI 호출
🔌
MCP Client
MCP 서버와 1:1 연결 담당.
→ McpSyncClient / McpAsyncClient
→ 서버 Tool 목록 캐시
→ 요청/응답 직렬화 처리
⚙️
MCP Server
실제 기능을 제공하는 서버.
→ Tool / Resource / Prompt 노출
→ 독립 프로세스 or HTTP 서버
→ 언어 무관하게 구현 가능
⚙️ MCP 핵심 기능 3종
MCP 서버가 클라이언트에 제공할 수 있는 세 가지 기능 유형. 각각 역할이 명확히 구분됨.
🛠️
Tool
AI가 직접 호출(실행)하는 함수.
→ 부수효과 있는 액션
→ DB 조회/수정, API 호출
→ LLM이 판단해서 호출 시점 결정
model-controlled action
📄
Resource
AI에게 제공하는 읽기 전용 컨텍스트.
→ 파일, DB 레코드, API 결과
→ URI로 식별 (file:// / db://
→ 앱이 언제 포함할지 결정
app-controlled read-only
💬
Prompt
재사용 가능한 프롬프트 템플릿.
→ 파라미터화된 메시지 구조
→ 멀티턴 대화 시나리오
→ 사용자 또는 앱이 선택
user-controlled template
🛠️ Tool — Spring AI 서버 구현
JAVA McpServer Tool 등록
// 1. JSON Schema 정의 String schema = """ { "type": "object", "properties": { "city": { "type": "string", "description": "도시명" } }, "required": ["city"] } """; // 2. Tool Specification 생성 var weatherTool = new McpServerFeatures.SyncToolSpecification( new Tool("get_weather", "현재 날씨 조회", schema), (McpSyncServerExchange exchange, Map args) -> { String city = (String) args.get("city"); return new CallToolResult(List.of( new TextContent(fetchWeather(city)) ), false); } ); // 3. McpServer에 등록 McpSyncServer server = McpServer.sync(transport) .serverInfo("weather-server", "1.0") .tools(weatherTool) // Tool 등록 .build();
📄 Resource — 컨텍스트 제공
JAVA Resource 등록
var schemaResource = new McpServerFeatures.SyncResourceSpecification( new Resource( "db://schema/users", // URI "users 테이블 스키마", // 이름 "DB 스키마 구조 정보", // 설명 "text/plain", null // mimeType ), (exchange, req) -> new ReadResourceResult(List.of( new TextResourceContents( req.uri(), "text/plain", getSchemaText() // 실제 내용 반환 ) )) );
💬 Prompt — 템플릿 제공
JAVA Prompt 템플릿 등록
var sqlPrompt = new McpServerFeatures.SyncPromptSpecification( new Prompt("sql-query-helper", "SQL 작성 도우미", List.of(new PromptArgument("table", "테이블명", true)) ), (exchange, req) -> { String table = req.arguments().get("table"); return new GetPromptResult("SQL 쿼리 작성 도우미", List.of( new PromptMessage(Role.USER, new TextContent("" + table + " 테이블 조회 쿼리를 작성해줘")) )); } );
🚌 Transport — STDIO vs SSE
MCP Client ↔ Server 간 통신 채널. 두 가지 방식 중 배포 환경에 맞게 선택.
STDIO
SSE (HTTP)
비교
⚡
STDIO Transport
표준 입출력 파이프로 통신 — 로컬 프로세스 전용
✅ 장점
→ 네트워크 설정 불필요
→ 오버헤드 최소 (매우 빠름)
→ 보안 설정 단순
→ 개발/테스트 환경 최적
⚠️ 제약
→ 동일 머신에서만 동작
→ 원격 서버 연결 불가
→ 다중 클라이언트 공유 어려움
→ 클라이언트가 서버 프로세스 관리
JAVASTDIO 서버
// 서버: STDIO로 기동 var transport = new StdioServerTransport(); McpSyncServer server = McpServer.sync(transport) .serverInfo("my-server","1.0") .tools(myTool) .build();

⚙️ Spring AI MCP 전체 구성
의존성 추가부터 ChatClient 연동까지 — 실제 동작하는 최소 구성 코드.
① 의존성 추가
클라이언트 / 서버 역할에 따라 다른 starter 사용.
XMLpom.xml
<!-- MCP 클라이언트 (Host 앱) --> <dependency> <groupId>org.springframework.ai</groupId> <artifactId>spring-ai-mcp-client-spring-boot-starter</artifactId> </dependency> <!-- MCP 서버 (Spring Boot 기반) --> <dependency> <groupId>org.springframework.ai</groupId> <artifactId>spring-ai-mcp-server-webmvc-spring-boot-starter</artifactId> </dependency>
② 클라이언트 application.yml
연결할 MCP 서버 목록을 설정. STDIO / SSE 모두 지원.
YAMLapplication.yml (클라이언트)
spring: ai: mcp: client: enabled: true toolcallback: enabled: true # Tool → ChatClient 자동 연동 connections: # STDIO 방식 weather-server: type: STDIO command: java args: ["-jar", "weather-mcp.jar"] # SSE 방식 search-server: type: SSE url: http://search-mcp:8080
③ ChatClient에서 MCP Tool 사용
ToolCallbackProvider가 자동 등록 → 별도 코드 없이 AI가 MCP Tool 호출 가능.
JAVAChatService.java
@Service public class ChatService { private final ChatClient chatClient; public ChatService( ChatClient.Builder builder, ToolCallbackProvider mcpTools // 자동 주입 ) { this.chatClient = builder .defaultTools(mcpTools) // MCP Tool 전체 등록 .build(); } public String ask(String question) { return chatClient.prompt() .user(question) .call() .content(); // AI가 필요시 MCP Tool 자동 호출 ✓ } }
④ MCP 서버 application.yml
서버 측 최소 설정. Tool Bean을 자동 감지.
YAMLapplication.yml (서버)
spring: ai: mcp: server: name: weather-mcp-server version: 1.0.0 type: SYNC # or ASYNC sse-message-endpoint: /mcp/message capabilities: tool-change-notification: true # 동적 Tool 변경 알림
🎯
핵심 요약: Spring AI의 toolcallback.enabled: true 옵션 하나로 MCP 서버의 모든 Tool이 ChatClient에 자동 연동됨. 개발자는 MCP 서버의 비즈니스 로직만 집중하면 됨.
