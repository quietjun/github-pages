01 · PEFT 란?
PEFT
Parameter Efficient Fine-Tuning  ·  파라미터 효율적 미세조정

사전학습된 대규모 언어 모델(LLM)을 특정 태스크에 맞게 적응시키는 기법.
전체 파라미터를 재학습하는 대신, 소수의 파라미터만 학습하여 시간·메모리·저장 공간을 대폭 절약.

02 · Low-Rank 란?
💡 핵심 발견: 사전학습된 LLM을 파인튜닝할 때, 가중치 변화(ΔW)는 낮은 내재 차원(low intrinsic dimension)을 가진다.
→ 실제 중요한 변화는 전체 차원이 아닌 훨씬 작은 부분공간에서만 일어남.
→ ΔW 전체를 학습할 필요 없이, rank가 작은 두 행렬 B × A로 근사해도 충분하다.
RANK 개념
Rank(계수)란?

행렬이 실제로 표현할 수 있는 선형 독립적인 방향의 수.
8×8 행렬의 full rank = 8이지만, 파인튜닝 시 ΔW의 실질적 변화는 rank=2 정도의 낮은 차원만으로도 충분히 근사 가능하다는 것이 실험적으로 밝혀짐.
→ LoRA는 이 원리를 활용해 큰 행렬 ΔW를 두 개의 작은 행렬 B × A로 분해하여 학습.

▶ 행렬 분해 시각화 — ΔW (8×8) ≈ B (8×2) × A (2×8)

LOW-RANK 근사
왜 작아도 되나?
특정 태스크 적응에 필요한 정보는 소수 방향에 집중됨.
→ 전체 차원 대신 중요 방향만 학습해도 성능 유지.
실제 절감 효과
대형 모델 스케일
d=4096, rank=8 기준:
· ΔW: 4096² = 16.7M
· LoRA: 4096×8 × 2 = 65K (99% 절약)
03 · 순전파 & 역전파 동작 방식
순전파 (Forward) 역전파 (Backward) 동결 (학습 안 함) 학습 대상
① 입력
② 병렬 계산
③ 합산 → 출력
④ 오차 계산
⑤ 동시에

순전파

x가 두 경로로 동시에 흘러 각각 계산 후 합산

동결

W₀는 사전학습 지식 보존을 위해 건드리지 않음

학습

B·A만 파라미터 업데이트 대상

출력

두 경로 합산 = 최종 예측값

역전파

오차 → Loss에서 두 갈래로 동시에 흐름

NO UPDATE

W₀: gradient 계산하지만 파라미터 변경 없음 → 보존

UPDATE

B, A: gradient 적용 → 태스크에 맞게 수정됨

04 · 트랜스포머 내 LoRA 적용 위치
🎯 Self-Attention Layer
q_proj Query 투영 LoRA ✔
k_proj Key 투영 선택적
v_proj Value 투영 LoRA ✔
o_proj Output 투영 선택적
⚙️ MLP (Feed-Forward) Layer
up_proj Up 투영(증폭) 선택적
gate_proj Gate 투영(검증) 선택적
down_proj Down 투영(안정화) 선택적
💡 일반적으로 q_proj + v_proj에만 적용해도 충분한 성능 달성 — 메모리 절감과 성능 간 균형이 가장 좋은 조합. 더 높은 성능이 필요하면 k_proj, o_proj까지 확장.

06 · 추론 방식 — 분리 vs 병합
🔗 분리 방식 (일반)
출력 = W₀·x + B·A·x
→ 두 번 계산 후 합산
  • LoRA Adapter 교체 가능
  • 원본 W₀ 항상 보존
  • 여러 태스크 즉시 전환
  • 계산 두 번 → 약간 느림
⚡ 병합 방식 (최적화)
W' = W₀ + B·A  (사전 합산)
출력 = W'·x
→ 한 번 계산으로 끝
  • 추론 속도 빠름 (오버헤드 없음)
  • 원본과 동일 구조 유지
  • 병합 후 Adapter 교체 불가
  • 단일 태스크 배포에만 적합
07 · LoRA 코드 사용법
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig, get_peft_model, TaskType
import torch

# ── 1. Base Model 로드 (가중치 동결 상태로 로드됨) ──────────────────────────
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    torch_dtype=torch.float16,
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-hf")

# ── 2. LoRA Config 설정 ──────────────────────────────────────────────────────
lora_config = LoraConfig(
    r=16,                              # rank: Low-Rank 행렬 크기 (작을수록 파라미터 ↓)
    lora_alpha=32,                    # scaling = alpha / r → 실제 학습 배율
    target_modules=["q_proj", "v_proj"],  # LoRA 적용 레이어 (q + v 기본 권장)
    lora_dropout=0.05,               # Dropout: 과적합 방지
    bias="none",                      # bias 파라미터는 학습 안 함
    task_type=TaskType.CAUSAL_LM       # 태스크 타입: 인과 언어모델
)

# ── 3. PEFT 모델 생성 → A, B 행렬만 학습 가능 상태로 전환 ─────────────────
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# 출력 예: trainable params: 4,194,304 / all params: 6,742,609,920 (0.06%)

# ── 4. Adapter 저장 & 로드 ─────────────────────────────────────────────────
model.save_pretrained("./lora-adapter")     # ~10MB Adapter만 저장

# ── 5. (선택) 추론용 병합 방식 ─────────────────────────────────────────────
merged_model = model.merge_and_unload()   # W' = W₀ + B·A → 단일 모델로 병합
merged_model.save_pretrained("./merged-model")
