01 · 양자화(Quantization) 란?
QUANTIZATION
양자화 — 모델 가중치를 더 적은 비트로 표현하는 압축 기법

부동소수점 숫자를 표현하는 비트 수를 줄여 모델이 차지하는 메모리를 줄이는 기법.
비트가 줄면 표현할 수 있는 숫자의 범위와 정밀도가 낮아지지만, 잘 설계된 양자화는 성능 손실을 최소화하면서 메모리를 대폭 절감.
→ QLoRA는 Base 모델을 4-bit(NF4)로 양자화해 로드하고, 그 위에 LoRA Adapter를 붙여 학습.

  • 부호 1 + 지수 8 + 가수 23비트
  • 표현 범위·정밀도 최대
  • 학습 기본값, 메모리 부담 큼
FP — Floating Point
  • 부동소수점 · 소수점 위치가 유동적
  • 부호 + 지수 + 가수 세 파트로 구성
  • 매우 크거나 작은 수 유연하게 표현
  • 딥러닝 기본 수 표현 형식
FP16 / BF16
  • FP32 대비 메모리 50% 절감
  • LoRA Adapter 정밀도
  • GPU Tensor Core 가속 지원
FP16 vs BF16
  • FP16 · 지수 5 + 가수 10bit → 정밀도 ↑, 표현 범위 ↓
  • BF16 · 지수 8 + 가수 7bit → FP32와 동일 표현 범위
  • BF = Brain Float · 구글 브레인 팀 설계
  • 학습 안정성 → BF16 선호 (오버플로우 ↓)
INT8
  • 정수 범위만 표현
  • 비선형 표현력 손실 있음
  • 주로 추론(inference)에 사용
INT — Integer
  • 정수(Integer) · 소수점 없음
  • 연산 속도 빠름 · 하드웨어 지원 우수
  • 연속 실수 표현 시 손실 발생
  • 학습보다 배포·추론 최적화에 적합
NF4 QLoRA
  • 정규분포 기반 비균일 양자화
  • INT4보다 정보 손실 최소화
  • QLoRA의 핵심 양자화 방식
NF — NormalFloat
  • QLoRA 논문 제안 포맷
  • 가중치 → 정규분포 형태로 분포
  • 값 밀집 구간에 양자화 구간 촘촘히 배치
  • INT4 대비 정보 손실 최소화 → 성능 유지

03 · QLoRA 핵심 기술 3가지
01 / NF4
NF4 NormalFloat
4-bit 양자화 방식
💡 왜 필요?
일반 INT4는 균일 간격으로 값을 배치 → 가중치가 몰려 있는 구간의 정밀도 낭비.
NF4는 정규분포를 가정해 가중치가 많은 구간에 더 촘촘하게 배치.
INT4 — 균일 간격
  • 구간 크기 동일
  • 중간값 구간 낭비
  • 극단값 정밀도 ↓
NF4 — 비균일 간격
  • 정규분포 맞춤 배치
  • 중간 밀집 구간 세밀
  • 정보 손실 최소화 ★
02 / DQ
Double Quantization
양자화 상수까지 재압축
💡 왜 필요?
NF4 양자화 시 각 블록마다 양자화 상수(scale factor)가 생성됨.
이 상수 자체도 메모리를 차지 → 상수까지 다시 양자화해 추가 절감.

03 / PO
Paged Optimizers
GPU → CPU 자동 스왑
💡 왜 필요?
학습 중 배치가 클 때 순간적으로 GPU 메모리 부족(OOM) 발생.
Optimizer 상태를 CPU RAM으로 자동 페이징해 OOM 방지.
✖ 일반 Optimizer
메모리 초과 → OOM 크래시
✔ Paged Optimizer
초과분 → CPU RAM 자동 스왑
  • NVIDIA 통합 메모리 기반 동작
  • 더 큰 배치 사이즈 허용

05 · 코드 사용법
# ── QLoRA의 "Q" 부분 — 4-bit 양자화 설정 ─────────────────────────────────
from transformers import BitsAndBytesConfig
import torch

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,                    # NF4로 4-bit 압축 로드 활성화
    bnb_4bit_quant_type="nf4",            # 양자화 방식: NF4 (INT4보다 정밀)
    bnb_4bit_compute_dtype=torch.bfloat16, # Forward 시 복원 정밀도: BF16
    bnb_4bit_use_double_quant=True,      # Double Quantization 활성화
)

# 각 옵션의 의미:
# load_in_4bit          → Base 모델을 NF4 4-bit으로 압축해 GPU에 적재
# bnb_4bit_quant_type   → "nf4": 정규분포 최적화 / "fp4": 부동소수점 4-bit
# bnb_4bit_compute_dtype→ 실제 연산(Forward)은 BF16으로 복원 후 수행
# bnb_4bit_use_double_quant → 양자화 상수도 재양자화 → 추가 ~0.5GB 절감
# ── 양자화된 모델 로드 + k-bit 학습 준비 ─────────────────────────────────
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import prepare_model_for_kbit_training

# 1. 4-bit 양자화된 상태로 모델 로드 (7B 기준 ~4GB)
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config,    # 위에서 정의한 4-bit 설정 적용
    device_map="auto",                # GPU/CPU 자동 배분
)
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-hf")

# 2. k-bit 학습을 위한 사전 준비
#    - Layer Norm을 FP32로 캐스팅 (학습 안정성)
#    - Gradient Checkpointing 활성화 (메모리 추가 절감)
#    - 입력 임베딩 gradient 활성화
model = prepare_model_for_kbit_training(model)

# 이 시점의 model:
# - Base 가중치: NF4 4-bit (동결)
# - LoRA Adapter: 아직 미부착 → 다음 단계에서 추가
# ── LoRA Adapter 부착 — QLoRA의 "LoRA" 부분 ──────────────────────────────
from peft import LoraConfig, get_peft_model, TaskType

lora_config = LoraConfig(
    r=64,                                  # rank: 메모리 여유로 크게 설정 가능
    lora_alpha=16,                         # scaling = alpha / r
    target_modules=[                       # LoRA 적용 레이어
        "q_proj", "v_proj",              # 기본 권장
        "k_proj", "o_proj",              # 성능 향상 시 추가
    ],
    lora_dropout=0.1,
    bias="none",
    task_type=TaskType.CAUSAL_LM
)

# Adapter 부착 — A, B 행렬만 FP16으로 학습 가능 상태
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# 출력 예: trainable: 0.85% | Base: NF4 4-bit | Adapter: FP16

# Adapter 저장 (~10MB)
model.save_pretrained("./qlora-adapter")
