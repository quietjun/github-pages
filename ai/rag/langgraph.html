01 LCEL vs LangGraph — 무엇이 다른가?

LCEL · 기존 방식

일방통행 레시피
→ 순서대로만 실행, 되돌아가기 불가

직선 구조 / 유턴 불가 단순 파이프라인에 최적 조건 분기·반복 처리 어려움

LangGraph · 새로운 방식

사거리·회전교차로가 있는 도로망
→ 조건 분기, 루프, 유턴 모두 가능

사이클(Cycle) 구조 지원 조건부 엣지로 동적 분기 복잡한 에이전트 로직 구현 가능
02 핵심 구성요소
📋

State

공유 메모장 · 공용 게시판
  • 모든 노드가 공유하는 데이터 저장소
  • 이전 작업 기록 → 에러 복구 가능
  • TypedDict / Pydantic 기반 정의
  • 중간 결과 누적 → 최종 판단 근거
⚙️

Node

전문가 팀원 · 담당자
  • 특정 업무만 수행하는 단위 함수
  • State 입력 → 처리 → State 업데이트
  • 검색 / 요약 / 번역 / 평가 등 분리
  • Python 함수 또는 LLM 호출 단위
🔀

Edge

업무 지시서 · 통로
  • 노드 간 연결 경로 정의
  • 일반 엣지: 항상 동일 경로 이동
  • 조건부 엣지: 상태에 따라 동적 분기
  • 품질 낮음 → 재검색 / 완료 → END

05 언제 무엇을 써야 할까?

LCEL 선택 시

  • 단순 번역 / 요약 등 1회성 작업
  • 입출력이 명확한 선형 파이프라인
  • 빠른 프로토타이핑 필요 시
  • 분기·반복 없는 단순 체인

LangGraph 선택 시

  • 논문 수십 편 읽고 비판적 분석 보고서
  • 코드 작성 → 에러 수정 → 재실행 루프
  • 다단계 검증이 필요한 RAG 파이프라인
  • 사람 승인이 필요한 자동화 에이전트
🤖

LangGraph = LLM을 진짜 '에이전트'로 격상시키는 도구

단순 챗봇 → 스스로 사고하고 행동하는 자율 에이전트로 진화
