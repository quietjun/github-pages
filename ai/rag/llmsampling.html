② Top-k · Top-p — 슬라이더 조절
K
🔢 Top-k
  • 확률 상위 k개 토큰만 후보로 유지 (고정 개수)
  • 분포 형태와 무관하게 항상 k개 유지
  • 범위: 1 ~ V (V = 어휘 크기)

📊 Top-p (Nucleus Sampling)
  • 누적 확률 합 ≥ p가 되는 최소 토큰 집합 유지
  • 분포가 뾰족하면 후보 少, 평평하면 후보 多 (적응적)
  • 범위: 0.0 ~ 1.0

④ 파라미터 조합 가이드 — 용도별 권장 설정

🎯 정확·일관 답변 Sharp

  • Temperature 낮게 → 분포 뾰족 → 1위 토큰 집중 선택
  • Top-k/p는 보조적 — Temperature만으로 충분한 경우 多
✅ 용도: 코드 생성 / 수학 계산 / 사실 Q&A / 번역 / 요약
⚙️ 권장: T=0.2, Top-p=0.9

🎨 창의적 글쓰기 Flat

  • Temperature 높게 → 분포 평평 → 다양한 표현 후보 확보
  • Top-p로 완전 엉뚱한 토큰만 걸러냄
✅ 용도: 소설 / 시 / 브레인스토밍 / 광고 카피
⚙️ 권장: T=1.0, Top-p=0.95, Top-k 비활성화

⚡ Top-k 단독 고정 개수

  • 분포 형태와 무관하게 항상 k개 → 예측 가능한 후보 수
  • 분포가 뾰족해도 k개 강제 유지 → 불필요 후보 포함 가능
✅ 용도: 출력 후보 수를 일정하게 유지해야 하는 시스템
⚙️ 권장: T=1.0, Top-k=40~50

🌊 Top-p 단독 적응적

  • 분포에 따라 후보 수 자동 조절 → 상황 적응적
  • Claude·GPT-4 등 최신 모델이 선호하는 방식
✅ 용도: 대화형 AI / 챗봇 / 일반 텍스트 생성
⚙️ 권장: T=1.0, Top-p=0.9 (가장 흔한 조합)
