Transformer & Self-Attention

학습된 Transformer가 텍스트를 입력받아 다음 단어를 예측하는 추론 과정

LSTM이 Transformer에 밀린 이유
⏳ LSTM
순차 처리 — 한 단어가 끝나야 다음으로 넘어간다
  • ⚠️순차 처리 — 앞 단어 완료 후 다음 처리
  • ⚠️병목 심화 — 문장이 길수록 학습 속도 저하
  • ⚠️장거리 의존성 희석 — Cell State로도 한계
  • ⚠️병렬화 불가 — GPU 다수여도 본질적 한계
⚡ Transformer
병렬 처리 — 모든 단어를 동시에 바라본다
  • ✅Self-Attention — 모든 단어 쌍 관계를 동시 계산
  • ✅완전한 병렬 처리 — GPU 최대 활용
  • ✅거리 무관 — 가깝든 멀든 동등하게 관계 파악
  • ✅현대 LLM의 핵심 — GPT, BERT, LLaMA 모두 Transformer 기반
📌
Positional Encoding — Transformer만의 필수 설계
· LSTM: 순차 처리 → 처리 순서 자체가 위치 정보
· Transformer: 병렬 처리 → 위치 정보가 사라짐
→ sin/cos 기반 위치 패턴을 임베딩에 명시적으로 추가
병렬 처리라는 구조적 선택이 만들어낸 필연적 요소
토큰 → 임베딩 벡터
🔵
임베딩(Embedding) — 단어를 고차원 벡터로 변환
· GPT-3 기준: 토큰 1개 = 12,288차원 벡터
· 비슷한 의미 → 벡터 공간에서 가까운 위치
· "왕 − 남자 + 여자 ≈ 여왕" 같은 벡터 연산 성립

💡
임베딩의 핵심: "먹었다"와 "먹다"의 벡터가 매우 비슷하다
Attention의 핵심 직관 — 임베딩을 문맥으로 업데이트한다
🟠
초기 임베딩 = 문맥 없는 단어 그 자체
· "모델" → 처음엔 패션모델인지 AI 모델인지 구분 불가
· Attention: 주변 단어를 참조해 "무엇을 더할지" 계산 → 임베딩 업데이트
동일한 단어, 다른 문맥 → 다른 임베딩
두더지 mole → mole = 동물
이산화탄소 mole → mole = 화학 단위
조직검사 mole → mole = 피부 점
처음 임베딩은 세 "mole" 모두 동일 — Attention 이후에야 의미가 분리된다.
Decoder-Only Transformer
📐
이 자료의 범위 — Decoder-only Transformer (GPT 계열)
원래 Transformer는 Encoder + Decoder 두 블록으로 구성된다 (번역 등에 사용).
· Encoder: 입력 문장 전체를 양방향으로 이해
· Decoder: Encoder 출력을 참조해 출력 문장을 순서대로 생성

GPT는 Decoder만 사용하는 구조이다. Encoder가 없으므로 Cross-Attention도 없다.
· 입력과 출력을 구분하지 않고 하나의 토큰 시퀀스로 처리
· Causal Masking으로 미래 토큰을 차단 → 왼쪽에서 오른쪽으로 순서대로 생성
· 이 자료는 이 Decoder-only 구조를 기준으로 설명한다
🟣
Self-Attention: 각 토큰이 문장 내 모든 토큰을 참조해 자신의 임베딩을 업데이트하는 연산
학습이 완료된 Transformer가 텍스트를 입력받아 다음 단어를 예측하기까지, 내부에서 일어나는 추론 과정이다.

②
Q · K · V 벡터 생성
학습된 가중치 행렬 W_Q, W_K, W_V를 임베딩에 곱해 세 벡터를 만든다
임베딩 → 3가지 선형변환 (학습된 상수 행렬)
Q = E · W_Q
K = E · W_K
V = E · W_V
3 토큰 × 3종류 = 총 9개 벡터
Query
Q₁, Q₂, Q₃
"나는 어떤 정보를 찾고 있나?"
Q₁: [0.6, -0.2, 0.4, …]
Q₂: [0.1, 0.7, 0.3, …]
Q₃: [0.4, 0.2, 0.8, …]
Key
K₁, K₂, K₃
"나는 어떤 정보를 가지고 있나?"
K₁: [0.5, 0.3, 0.1, …]
K₂: [0.2, 0.6, 0.4, …]
K₃: [0.4, 0.1, 0.7, …]
Value
V₁, V₂, V₃
"실제로 전달할 정보"
V₁: [0.3, 0.5, 0.2, …]
V₂: [0.7, -0.1, 0.4, …]
V₃: [0.2, 0.6, 0.8, …]
💡
W_Q, W_K, W_V: 사전 학습으로 확정된 가중치 행렬
· 이 행렬 한 세트(= Single Head)가 한 가지 관계 패턴을 포착하도록 학습된다
· Single Head만으로는 문장의 다양한 관계를 동시에 포착하기 어렵다 → Multi-Head Attention의 필요성
· 예) 어떤 세트는 "주어-동사 관계"를, 다른 세트는 "시제-동사 관계"를 주로 포착
③
Score 계산 — Q · Kᵀ / √d_k
각 Query와 모든 Key의 내적(dot product)으로 "얼마나 관련 있는지" Score를 계산한다
score(Qᵢ, Kⱼ) = Qᵢ · Kⱼᵀ / √d_k
d_k: Q · K 벡터의 차원 수 (GPT-3 기준 128)
예: Q₁("나는")과 각 Key의 유사도를 계산한다
Raw Score (softmax 이전)
K₁ 나는 K₂ 밥을 K₃ 먹었다
Q₁ 나는 3.20.82.1
Q₂ 밥을 0.42.43.8
Q₃ 먹었다 1.93.13.5
왜 √d_k로 나누는가
· d_k가 크면 내적값이 지수적으로 커짐
· 나누지 않으면 softmax 후 한 값만 1에 가까워져 다양성 소실
④

각 행의 합이 1.0이 되도록 변환 → 어떤 토큰에 얼마나 "집중"할지 결정된다

Causal Masking (GPT · Decoder 계열)
현재 토큰이 자신보다 앞에 있는 토큰만 참조하도록 미래 위치의 Score를 -∞로 차단하는 기법

· 미래 토큰의 score → -∞ 설정 → softmax 후 0
· "밥을"은 "먹었다"를 절대 참조 불가
· 추론 시 미래 정답 노출 방지

➡️
Attention Weight × Value = 가중합
· Attention Weight: 각 토큰을 얼마나 참조할지의 비율
· Value: 각 토큰이 전달할 정보 벡터
· 가중합: 이 비율대로 Value들을 섞어 하나의 벡터로 만드는 연산 → 다음 Step에서 임베딩에 더한다
⑤
가중합 → 임베딩 업데이트
Attention Weight × V 벡터를 가중합해 각 토큰의 임베딩에 더한다
Output = Attention Weight × V  →  원래 임베딩에 더함 (Residual)

Multi-Head Attention — 여러 관점을 병렬로
🟣
Multi-Head Attention — 여러 관계 패턴을 병렬로 포착
· 단일 Head: W_Q·W_K·W_V 한 세트 → 한 가지 관계 패턴만 학습
· Q·K·V를 서브스페이스로 분할 → Head 수만큼 병렬 수행 → 각 Head가 서로 다른 관계 패턴 학습
· GPT-3: 96개 Head, 각 Head 차원 = 128
🔴
Head 1
시간 맥락 관계
"오늘 ↔ 점심"
🔵
Head 2
목적어 관계
"뭐 ↔ 먹을까"
🟢
Head 3
의문문 구조
"먹을까 ↔ ?"
🟣
Head 4~N
기타 관계 패턴
"오늘 ↔ 먹을까 …"
📌
위 레이블("주어-동사", "시제-동사" 등)은 사람이 미리 지정한 것이 아니다.
· 설계 시: Head 개수(예: 96개)만 결정
· 학습 후: 각 Head의 W_Q·W_K·W_V가 어떤 관계를 포착할지는 학습이 스스로 결정
· 연구자들이 학습 완료 후 각 Head의 Attention 패턴을 사후 분석해 명명한 것
