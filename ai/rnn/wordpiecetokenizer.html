학습되지 않은 빈 토크나이저 생성 → WordPiece 알고리즘으로 서브워드 분리 과정 시각화

🗂️ Tokenizer 종류 및 역할
BertWordPieceTokenizer
WordPiece 알고리즘으로 단어를 서브워드로 분리
단어 내 연속 서브워드에는 ## 접두사 부착 → 단어 시작 위치 구분
[CLS] [SEP] 등 특수 토큰 함께 사용
📚 훈련(train) 파라미터
vocab_size default: 30,000
WordPiece 학습 완료 후 최종 토큰 목록의 총 크기
자주 등장하는 서브워드 쌍을 병합하다가 이 수에 도달하면 학습 종료.
  • 너무 작으면 → 병합 조기 종료 → 짧은 서브워드·문자 단위 토큰 다수 잔존 → 시퀀스 길어짐 → 연산량 증가
  • 너무 크면 → 희귀 단어도 vocab에 포함, 메모리·학습 비용 증가
  • BERT 기본값 30,000 : 영어 기준 균형점 / 한국어는 형태소 다양성으로 더 큰 값 사용 권장
initial_alphabet
학습 전에 vocab에 미리 추가해 둘 문자(character) 집합
  • 학습 데이터에 등장하지 않아도 반드시 포함시켜야 하는 문자 지정 가능
  • 특수 토큰([CLS], [SEP] 등)은 별도 special_tokens 인자로 추가 → initial_alphabet과 구분됨
limit_alphabet default: 1,000
base vocab(문자 단위 초기 vocab)에 등록될 문자의 최대 개수 상한선
빈도가 낮은 문자부터 제외 → 제외된 문자는 추론 시 [UNK]로 처리될 수 있음
min_frequency default: 2
서브워드 쌍(pair)을 병합할 때 적용되는 최소 등장 빈도 조건
  • 인접 서브워드 쌍이 이 값 이상 등장해야만 병합 후 vocab에 추가됨
  • 기본값 2 : 딱 1번만 붙어서 등장한 쌍은 병합 안 됨
  • 개별 문자(base vocab)에는 미적용 → 학습 데이터에 단 1번 등장해도 문자는 vocab에 무조건 등록
special_tokens
특수 토큰 목록 직접 지정. BERT 기본 5종:
  • [PAD] : 배치 처리 시 시퀀스 길이를 맞추기 위한 패딩 토큰
  • [UNK] : vocab에 없는 OOV 단어를 대체하는 토큰
  • [CLS] : 문장 시작 / 분류(Classification) 태스크에서 문장 전체 표현으로 활용
  • [SEP] : 두 문장 사이 구별자 / 문장 경계 표시
  • [MASK] : MLM 학습 시 마스킹된 위치를 나타내는 토큰
wordpiece_prefix default: '##'
단어 내 첫 번째 위치가 아닌 연속 서브워드임을 표시하는 접두사
  • "playing" → play + ##ing
  • "unbelievable" → un + ##believ + ##able
  • 첫 서브워드는 ## 없음 → 단어 시작 위치 구분 가능

⚖️ 원핫 인코딩 vs nn.Embedding
토큰 ID를 밀집 벡터로 변환하는 두 방식 비교 — BERT는 nn.Embedding 사용
❌ 원핫 인코딩
고양이가 (ID=1523)
→ [0, 0, …, 1, …, 0]
(30,000차원)
→ vocab 크기만큼 차원 폭발 → 대부분 0 → 비효율적 → 단어 간 의미 관계 표현 불가
✅ nn.Embedding
고양이가 (ID=1523)
→ [0.12, -0.34, 0.88, …]
(768차원, 밀집 벡터)
→ 고정된 작은 차원 (보통 768) → 모든 값이 의미를 담음 → 학습을 통해 의미 관계 반영
📖 nn.Embedding 동작 원리
수학적으로는 원핫 × 가중치행렬과 동일하지만,
실제 구현은 인덱스로 행을 직접 꺼내는 룩업 테이블 → 효율적이고 빠름
쉽게 말하면 map[token_id] → 벡터를 바로 반환하는 구조
코드 예시
import torch
import torch.nn as nn

embedding = nn.Embedding(30000, 768)
token_ids = torch.tensor([5, 1523, 892, 6])
# [CLS] 고양이가 좋아요 [SEP]
vectors = embedding(token_ids)
# → shape: [4, 768] (토큰 수 × 임베딩 차원)

🤖 BERT 학습 — MLM (Masked Language Modeling)
BERT는 입력 토큰 시퀀스에서 일부를 [MASK]로 가리고,
주변 문맥을 보고 원래 토큰을 맞추는 방식으로 언어 표현을 학습합니다.
① 원본 문장 → 토크나이저 변환
"고양이가 생선을 먹어요"
⚠️ 아래는 "생선을", "먹어요"가 vocab에 미등록된 상황을 가정한 예시입니다.
실제로 충분히 학습된 BERT 토크나이저라면 이 단어들이 단어 단위로 vocab에 포함될 수 있습니다.
고양이가 생 ##선 ##을 먹 ##어 ##요
Token IDs: [1523, 8821, 4532, 221, 9012, 334, 219]
② 랜덤 15% 마스킹 → "생"(ID: 8821) 토큰을 [MASK]로 대체
고양이가 [MASK] ##선 ##을 먹 ##어 ##요
입력 IDs : [1523, 4, 4532, 221, 9012, 334, 219]
정답 label : [-100, 8821, -100, -100, -100, -100, -100]
→ -100 : 손실 계산 제외 / 8821 : "생"의 vocab ID
📈 BERT 예측 & 가중치 업데이트
BERT가 문제 풀기
📥 입력 : [고양이가] [MASK] [##선] [##을] [먹] [##어] [##요]
🧠 "고양이가" 다음 + "##선"과 함께 등장 + "먹어요"와 연관
    → vocab 30,000개 각각에 대해 확률 계산 (softmax)
ID 1 : "고" → 0.001% ID 2 : "양" → 0.003% … ID 8821 : "생" → 85.2% ✅ ID 8822 : "선" → 2.1%
✅ 맞혔을 때
예측 8821("생") = 정답 8821("생") → Loss 낮음
→ 100% 확률이 아니므로 약간의 가중치 조정 / 다음엔 더 확신 있게 예측
❌ 틀렸을 때
예측 4532("선") ≠ 정답 8821("생") → Loss 높음
→ 역전파로 가중치 크게 업데이트
→ "고양이가 다음 + ##선과 함께 등장 시 '생'이 올 확률 높다" 학습
🚀 이 과정을 수백만 번 반복 → 감정 분석, 질의응답, 문장 분류 등 다양한 NLP 태스크에 파인튜닝 가능한 모델 완성
